Capstone Work

Projects

Project 01  ·  SDG 7

EnergyBuddy

Problem Statement

Many households are unaware of how much electricity their appliances consume daily, leading to high bills and unnecessary energy waste. EnergyBuddy helps users track appliance usage across six screens aligned with SDG 7.

Project 1 taught me how to architect a multi-screen Compose app using a shared AppViewModel and StateFlow. I learned to connect UI state across screens without prop drilling. Designing for SDG 7 made me think beyond functionality to real-world energy impact.

Project 02  ·  SDG 7

EnergyBuddy — Full Stack

Problem Statement

Building on Project 1, users needed cloud-synced data, real-time weather-based tips, and a smart way to add appliances without manual wattage lookup. Project 2 adds Firebase Auth, Firestore, a live weather API, and CameraX + ML Kit scanning.

Integrating four technical pillars in one app taught me to manage complex build configurations — resolving Firebase import conflicts, KSP Gradle issues, and Room entity registration errors. I learned to implement a dual-write strategy (Room-first, then Firestore sync) for reliable offline-first data. CameraX with ML Kit was the most exciting part, scanning real appliance labels and extracting wattage automatically.

Growth

Reflection & Learning Journey

Starting from Lab 1 where I built my first Jetpack Compose screen, I gradually developed confidence with Android architecture, navigation, and state management. The jump from Project 1 to Project 2 was significant — adding Room, Firestore, Retrofit, and CameraX all at once required careful debugging and a deeper understanding of how Android components interact. I learned that building a real app is less about writing new code and more about understanding error messages, dependency conflicts, and architectural decisions. Most importantly, tying everything to SDG 7 reminded me that software engineering is most meaningful when it solves real problems for real people.
